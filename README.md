# NICE PH

NICE PH(NICE Promotion Hub)는 소상공인이 자신의 업종과 사업 정보를 입력해 고객에게 보여줄 사업 랜딩페이지를 쉽게 만드는 서비스입니다. 복잡한 기능보다 복잡한 홍보 업무를 단순하게 만드는 것을 목표로 합니다.

## V1.1 목적

V1.1은 서버, 로그인, 데이터베이스, 실제 AI API 없이 브라우저에서 바로 실행되는 프론트엔드 프로토타입입니다. V1의 실시간 편집 기능을 유지하면서, 작성한 랜딩페이지를 고객 관점의 독립된 전체 화면으로 확인할 수 있습니다.

## 실행 방법

별도 설치나 빌드 과정이 없습니다.

1. 저장소의 `index.html`을 Chrome, Edge, Safari 등 최신 브라우저에서 엽니다.
2. **내 사업 페이지 만들기** 버튼을 눌러 입력 화면으로 이동합니다.
3. 직접 정보를 입력하거나 **예시 데이터 불러오기**로 NICE 여성의류 예시를 확인합니다.
4. **완성 페이지 보기**를 누르면 편집 도구 없이 랜딩페이지가 전체 화면으로 열립니다.
5. 완성 페이지의 **페이지 링크 복사**로 현재 브라우저 주소를 복사하거나 **편집으로 돌아가기**로 입력 화면에 복귀합니다.

완성 페이지는 URL의 `#preview` 해시로 구분되므로 새로고침해도 같은 보기 상태가 유지됩니다.

브라우저 보안 설정 때문에 로컬 파일 기능이 제한되는 환경에서는 간단한 정적 파일 서버를 사용할 수도 있습니다.

```bash
python -m http.server 8000
```

그 후 `http://localhost:8000`에 접속합니다.

## 현재 구현 기능

- 8개 업종 선택 및 업종별 기본 한 줄 소개 자동 제안
- 상호명, 한 줄 소개, 사업 소개 입력과 실시간 미리보기
- 대표 이미지 및 최대 3개 상품·서비스 이미지 업로드 미리보기
- 최대 3개 상품·서비스와 3개 사업 강점 입력
- 주소, 영업시간, 전화번호, 상담·예약·쇼핑몰·인스타그램 링크 입력
- 입력된 정보와 유효한 링크만 랜딩페이지에 조건부 표시
- 데스크톱 좌우 편집 화면과 모바일 세로 배치
- 모바일 랜딩페이지 하단 고정 문의 버튼
- `localStorage` 자동 저장 및 새로고침 후 복원
- NICE 여성의류 예시 데이터 불러오기
- 전체 입력 초기화
- 키보드 포커스, 상태 메시지 등 기본 접근성 지원
- `#preview` 기반 고객 관점 전체 화면 완성 페이지
- 완성 페이지에서 편집 화면 복귀 및 현재 주소 복사
- 링크 복사 성공·실패 상태 안내

> 업로드 이미지는 브라우저 저장 공간에 Data URL 형태로 저장됩니다. 매우 큰 이미지는 브라우저 저장 한도를 초과할 수 있으며, 이 경우 화면 미리보기는 가능하지만 새로고침 후 복원되지 않을 수 있습니다.

## 파일 구조

```text
nice-ph/
├── index.html   # 시작 화면, 입력 폼, 랜딩페이지 미리보기 구조
├── style.css    # CSS 변수 기반 디자인과 반응형 스타일
├── app.js       # 상태 관리, 실시간 렌더링, 이미지 및 저장 기능
└── README.md    # 프로젝트 및 실행 안내
```

## V1.1 공유 링크의 현재 한계

- **페이지 링크 복사**는 현재 열려 있는 브라우저 주소만 복사합니다.
- 사업 데이터와 업로드 이미지는 사용자의 브라우저 `localStorage`에만 저장되며 URL에는 포함되지 않습니다.
- 따라서 복사한 주소를 다른 사람이나 다른 기기에서 열어도 작성한 사업 정보가 전달되지 않습니다.
- 로컬에서 `index.html`을 직접 연 경우 복사되는 주소는 `file://` 주소이므로 외부에서 접근할 수 없습니다.
- 실제 사용자별 공유 URL은 향후 서버 저장과 배포 기능이 추가되어야 제공할 수 있습니다.

## 향후 계획

V1 이후에는 업종별 테마, 실제 AI를 활용한 문구 생성, 계정 및 여러 페이지 관리, 게시·도메인 연결, 고객 반응 분석 등을 단계적으로 검토할 수 있습니다. 이 항목들은 현재 V1에는 포함하지 않습니다.

## V1.1 완료 기준 및 확인 방법

1. 모든 업종 버튼이 선택되는지 확인합니다.
2. 각 입력값이 오른쪽(모바일에서는 아래) 미리보기에 즉시 반영되는지 확인합니다.
3. 대표 이미지와 상품 이미지를 업로드해 미리보기 변화를 확인합니다.
4. 전화번호와 각 링크를 하나씩 입력·삭제하며 해당 행동 버튼만 표시되는지 확인합니다.
5. 새로고침 후 입력값이 복원되는지 확인합니다.
6. 예시 데이터와 전체 초기화 버튼을 확인합니다.
7. 브라우저 창 폭을 줄여 모바일 배치와 고정 문의 버튼을 확인합니다.
8. **완성 페이지 보기**를 눌러 입력 폼, STEP, 예시 데이터, 초기화 버튼이 사라지는지 확인합니다.
9. 완성 페이지가 데스크톱과 모바일에서 전체 폭으로 자연스럽게 표시되는지 확인합니다.
10. `#preview` 상태에서 새로고침한 뒤 완성 페이지가 유지되는지 확인합니다.
11. **페이지 링크 복사**의 성공 메시지를 확인하고, 복사가 제한된 브라우저에서는 실패 안내가 표시되는지 확인합니다.
12. **편집으로 돌아가기**를 눌러 기존 입력값이 유지된 편집 화면으로 돌아오는지 확인합니다.

위 항목이 모두 정상 작동하면 NICE PH V1.1 작업이 완료된 것으로 봅니다.
