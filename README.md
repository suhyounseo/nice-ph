# NICE PH

NICE PH(NICE Promotion Hub)는 소상공인이 자신의 업종과 사업 정보를 입력해 고객에게 보여줄 사업 랜딩페이지를 쉽게 만드는 서비스입니다.

> 기능을 많이 만드는 것이 목표가 아니라, 소상공인이 5분 안에 자신의 사업을 소개할 수 있도록 입력 경험을 최대한 단순하게 만들어 주세요.

## V1.2 목적

V1.2는 새로운 서비스 범위를 추가하는 버전이 아니라 실제 사용 과정에서 발견한 입력 불편을 개선한 UX 버전입니다. V1.1의 실시간 편집, 전체 화면 완성 페이지, 이미지 업로드, 브라우저 자동 저장 기능을 유지합니다.

## 실행 방법

별도 설치나 빌드 과정이 없습니다.

1. `index.html`을 Chrome, Edge, Safari 등 최신 브라우저에서 엽니다.
2. **내 사업 페이지 만들기**를 눌러 입력 화면으로 이동합니다.
3. 직접 정보를 입력하거나 **예시 데이터 불러오기**로 NICE 여성의류 예시를 확인합니다.
4. **완성 페이지 보기**를 누르면 편집 도구 없이 랜딩페이지가 전체 화면으로 열립니다.
5. 완성 페이지의 **공유**에서 현재 저장 방식에 대한 안내를 확인하거나 **편집으로 돌아가기**로 복귀합니다.

로컬 파일 실행이 제한되는 환경에서는 다음과 같이 정적 파일 서버를 사용할 수 있습니다.

```bash
python -m http.server 8000
```

그 후 `http://localhost:8000`에 접속합니다.

## V1.2 사용 방법

### 상품·서비스

- 처음에는 기존과 같은 3개 입력 카드가 표시됩니다.
- **상품 추가**로 필요한 만큼 카드를 추가합니다.
- 각 카드의 위·아래 버튼으로 순서를 바꾸고 **삭제**로 제거합니다.

### 강점

- 처음에는 3개 강점 입력란이 표시됩니다.
- **강점 추가**로 필요한 만큼 추가하고 각 항목의 **삭제**로 제거합니다.

### 고객 행동 버튼

- **버튼 추가**를 누른 뒤 버튼명, 링크, 선택 아이콘을 입력합니다.
- 쇼룸, 스마트스토어, 예약, 카카오채널, 블로그, 유튜브, 인스타그램, 자사몰 등 원하는 이름을 사용할 수 있습니다.
- `https://`, `tel:`, `mailto:` 링크를 지원하며 입력된 유효한 링크만 완성 페이지에 표시됩니다.

### 이미지

- 대표 이미지 한 장과 서브 이미지 여러 장을 등록할 수 있습니다.
- 대표 이미지가 Hero에 표시되며, 대표 이미지가 없으면 첫 번째 서브 이미지가 Hero로 사용됩니다.
- Hero에 사용되지 않은 서브 이미지는 사업 소개 아래 갤러리에 표시됩니다.

### 영업시간과 AI 인터뷰

- 영업시간은 여러 줄로 자유롭게 입력할 수 있습니다.
- **AI 인터뷰 준비중** 버튼은 향후 기능의 위치만 안내하며 현재는 작동 기능이 없습니다.

## 현재 구현 기능

- 8개 업종 선택과 업종별 기본 소개 문구
- 사업 정보 입력 즉시 랜딩페이지 반영
- 상품·서비스 무제한 추가, 삭제, 순서 변경
- 강점 무제한 추가 및 삭제
- 사용자 정의 행동 버튼 추가, 삭제, 순서 변경 및 선택 아이콘
- 대표 이미지와 다중 서브 이미지, 상품 이미지 업로드
- 자유 형식 다중 행 영업시간
- `localStorage` 자동 저장, 복원 및 V1/V1.1 데이터 자동 변환
- NICE 예시 데이터와 전체 초기화
- `#preview` 기반 고객 관점 전체 화면
- 공유 방식 안내와 현재 주소 복사
- 데스크톱·모바일 반응형 화면

## 코드 구조

```text
nice-ph/
├── index.html       # 입력 폼, 반복 항목 컨테이너, 미리보기와 공유 안내
├── style.css        # 기존 디자인 시스템, 반복 카드와 반응형 화면
├── app.js           # 저장 모듈, 입력 편집기, 미리보기와 화면 이벤트
└── README.md        # 프로젝트 사용 및 테스트 안내
```

`app.js`의 데이터 계층은 `load`, `save`, `clear` 규약의 저장 어댑터로 분리되어 있습니다. V2에서 서버 저장을 추가할 때 같은 규약의 서버 어댑터로 교체할 수 있으며 화면 로직은 변경하지 않아도 됩니다.

## 공유 기능의 현재 한계

- **공유**를 누르면 현재 브라우저 저장 방식이라는 안내가 먼저 표시됩니다.
- 현재 주소 복사는 같은 기기에서 다시 열어보기 위한 보조 기능입니다.
- 사업 정보와 이미지는 URL이 아니라 현재 브라우저의 `localStorage`에만 저장됩니다.
- 다른 사람이나 다른 기기에서는 복사된 주소만으로 작성 데이터가 전달되지 않습니다.
- `file://` 주소는 외부에서 접근할 수 없습니다.
- 이미지가 매우 많거나 크면 브라우저의 저장 용량 한도를 초과할 수 있습니다.
- 실제 사용자별 공유 URL은 V2 서버 저장 기능에서 지원할 예정입니다.

## 현재 제외 범위

서버 배포, 사용자별 공유 URL, 로그인, 데이터베이스, 실제 AI 인터뷰 및 AI API는 V1.2에 포함하지 않습니다.

## V1.2 테스트 방법

1. 기존 V1.1 저장 데이터가 복원되는지 확인합니다.
2. 상품을 네 개 이상 추가하고 삭제 및 위·아래 순서 변경을 확인합니다.
3. 강점을 네 개 이상 추가하고 삭제해 미리보기에 즉시 반영되는지 확인합니다.
4. 이름과 링크가 다른 행동 버튼을 여러 개 추가하고 아이콘 및 순서를 확인합니다.
5. 영업시간을 여러 줄로 입력해 줄바꿈이 유지되는지 확인합니다.
6. 대표 이미지와 여러 서브 이미지를 등록하고 Hero 및 갤러리를 확인합니다.
7. 새로고침 후 반복 항목, 이미지, 순서가 복원되는지 확인합니다.
8. 완성 페이지에서 **공유**를 눌러 V2 안내와 주소 복사 성공·실패 메시지를 확인합니다.
9. `#preview` 새로고침과 **편집으로 돌아가기**를 확인합니다.
10. 데스크톱과 모바일 너비에서 추가·삭제 버튼과 완성 페이지를 확인합니다.
