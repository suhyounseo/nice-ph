# NICE PH

NICE PH(NICE Promotion Hub)는 소상공인이 사업정보와 사진을 바탕으로 고객용 랜딩페이지와 채널별 홍보 콘텐츠를 만드는 브라우저 기반 도구입니다.

> 기능을 많이 만드는 것이 목표가 아니라, 소상공인이 5분 안에 자신의 사업을 소개할 수 있도록 입력 경험을 최대한 단순하게 만듭니다.

## V1.3

V1.3은 기존 랜딩페이지 생성 기능에 실제 홈페이지 게시판과 템플릿형 홍보센터를 연결합니다. 서버나 실제 AI·SNS API 없이 브라우저에서 바로 사용할 수 있으며 모든 데이터는 `localStorage`에 저장됩니다.

## 실행 방법

별도 설치 없이 `index.html`을 최신 Chrome 또는 Edge로 열 수 있습니다. 클립보드와 파일 기능을 안정적으로 확인하려면 프로젝트 폴더에서 정적 서버를 실행합니다.

```bash
python -m http.server 8000
```

그 다음 `http://localhost:8000`에 접속합니다.

## 주요 기능

- 업종·사업 소개·영업시간 입력과 실시간 고객 페이지 미리보기
- 대표·갤러리·상품 이미지 자동 리사이즈 및 압축
- 이미지별 `cover`, `contain`, 원본 비율 선택
- 상품과 강점 자유 추가·삭제, 상품 순서 변경
- 전화, 카카오, 네이버, Instagram 등 18종 행동 버튼과 자동 아이콘
- 홈페이지 게시물 작성·수정·삭제·상세 보기·카테고리 노출
- Instagram, Threads, Facebook, X, 네이버 블로그, 홈페이지용 템플릿 문구 생성
- 채널 문구 수정·재생성·복사·채널 열기
- 홍보글을 홈페이지 게시판에 즉시 등록
- 홍보글 보관함의 다시 열기·복제·삭제·채널별 일괄 복사
- 데모 댓글과 AI 답변 초안 복사
- 브라우저 자동 저장, V1.2 데이터 마이그레이션, JSON 백업·복원
- 작성 데이터를 포함하는 공유 링크와 대용량 이미지 제외 안전장치

## 홍보센터 사용 방법

1. 상단 `홍보센터`에서 목적, 대상 고객, 핵심 내용, 장점과 말투를 입력합니다.
2. `채널별 홍보글 만들기`를 누릅니다.
3. 채널 카드에서 다시 생성하거나 직접 수정하고 복사합니다.
4. `채널 열기`는 문구를 먼저 복사하고 플랫폼을 새 창으로 엽니다.
5. `홈페이지 게시판에 등록`에서 카테고리·제목·공개 상태를 확인해 저장합니다.
6. 계속 사용할 문구는 `홍보글 보관함에 저장`합니다.

이 생성기는 실제 AI가 아니라 입력 데이터와 사업정보를 조합하는 결정형 템플릿입니다.

## 홈페이지 게시판 등록 방법

- 상단 `게시물 관리`에서 카테고리, 제목, 본문, 이미지, 작성일, 연결 버튼과 공개 상태를 입력합니다.
- 저장하면 공개 게시물이 사업 랜딩페이지의 `새로운 소식` 영역에 최신순으로 표시됩니다.
- 처음 3개만 노출되고 `소식 더 보기`에서 전체 게시물을 확인합니다.
- 홍보센터에서 만든 문구도 각 채널 카드의 등록 버튼으로 같은 게시판에 저장할 수 있습니다.

## 데이터와 공유 한계

- 데이터와 이미지는 현재 브라우저의 `localStorage`에 저장됩니다.
- 공유 링크는 작성 데이터를 주소에 포함하지만 URL이 너무 커지면 이미지를 제외하고 안내합니다.
- 서버 저장 전에는 대용량 이미지가 포함된 완전한 타 기기 공유를 보장하지 않습니다.
- 중요한 작업은 `JSON 백업`으로 파일을 내려받아 보관하세요.
- 실제 SNS 자동 게시, 예약 발행, 외부 댓글 수집은 플랫폼 로그인과 API가 필요해 구현하지 않았습니다.

## 테스트 방법

1. NICE 예시를 불러온 뒤 새로고침하고 입력값 복원을 확인합니다.
2. 큰 사진과 투명 PNG를 올리고 압축 전·후 용량 안내와 비율 옵션을 확인합니다.
3. 상품·강점·행동 버튼을 추가하고 고객 페이지 반영을 확인합니다.
4. 게시물을 작성·수정·삭제하고 랜딩페이지·전체 소식·상세 화면을 확인합니다.
5. 홍보글 6채널 생성, 재생성, 수정, 복사, 채널 열기와 게시판 등록을 확인합니다.
6. 홍보글 보관함과 데모 댓글 답변 복사를 확인합니다.
7. JSON 백업 후 초기화하고 파일을 다시 불러옵니다.
8. 완성 페이지, 링크 복사, 새로고침 후 `#preview` 유지를 확인합니다.
9. 브라우저 개발자 도구의 모바일 화면과 콘솔 오류를 확인합니다.


## 현재 제외 범위

실제 AI API, SNS 자동 게시 API, 예약 발행 서버, 플랫폼 계정 연결, 외부 댓글 수집, 로그인, 데이터베이스, 결제는 V1.3 범위에 포함하지 않습니다.
